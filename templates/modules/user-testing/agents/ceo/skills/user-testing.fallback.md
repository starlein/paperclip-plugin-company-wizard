# Skill: User Testing (Fallback)

The UX Researcher primarily owns usability evaluations. You are the fallback — step in only if the researcher and PO are both absent or haven't started testing.

## User Testing (Fallback)

1. If no `docs/USER-TESTING.md` exists and no one has started:
   - Create a basic heuristic checklist covering the core user flow
   - Identify the top 3-5 usability concerns based on product goals
   - Document in `docs/USER-TESTING.md`
   - Mark all findings as **provisional** — they need validation by a researcher or PO
2. If the researcher or PO is active, skip this entirely.

## Rules

- This is a safety net. Keep it brief — enough to flag obvious issues.
- Let the researcher or PO own ongoing usability evaluation.
